# 🧬 VitaCore — Digital Twin OS

**VitaCore** is a high-fidelity, full-stack **"Digital Twin OS"** designed to aggregate, optimize, and gamify life's core telemetry metrics: **Health**, **Finance**, and **Career**.

Imagine the concept of a "Digital Twin" used in advanced engineering (where a virtual replica of a jet engine or smart factory is created to run simulations, monitor stresses, and predict issues before they happen). **VitaCore applies this exact engineering principle to human life.** By capturing daily biometric signals, financial assets, and professional velocity, VitaCore compiles a virtual clone of your profile to simulate future trajectories, establish routine presets, and deliver diagnostic insights.

---

## 🚀 How to Explain This Project (The Quick Pitch)

> *"VitaCore is a personal Digital Twin platform. Instead of tracking your life in separate, disconnected apps, VitaCore unifies your Health, Wealth, and Career into a single, high-fidelity virtual replica of yourself. Through our AI Simulation Engine, you can dial in sliders—such as study hours, savings rates, and sleep cycles—to run future forecasts (e.g. projecting how your current habits will impact your overall health score and net capital over the next six months) so you can make optimal choices in the physical world today."*

## 🛠️ Tech Stack & System Architecture

This project is built using a modern full-stack decoupled architecture:

```
    Vite + React 18 + TS Frontend 
               │ (REST APIs)
               ▼
     Express + Node.js Backend 
               │ 
               ├──► MongoDB Atlas (Data Ledger)
               │
               └──► OpenRouter Multi-Model Cloud (Fallback AI Loop)
                     ├── Attempt 1: Google Gemini 2.5 Flash
                     ├── Attempt 2: Llama 3 8B Instruct
                     └── Attempt 3: OpenAI GPT-3.5 Turbo
```

### 1. Frontend (`/frontend`)
* **Core Framework**: React 18 with TypeScript and Vite for lightning-fast hot-reloading and modular code.
* **Styling & Theme**: Tailwind CSS with custom backdrop filters, premium glassmorphism utility classes (`.glass-card` and `.neon-border`), and an immersive dark spaceship color palette.
* **Animations**: Framer Motion for smooth, organic page transitions and interactive micro-animations.
* **Charts & Telemetry**: Recharts wrapped with customized `ChartContainer` components for gorgeous weekly Sleep Architecture Area charts and Metabolic Balance Pie Wheels.

### 2. Backend (`/backend`)
* **Server Framework**: Node.js & Express serving fast, lightweight JSON APIs.
* **Database**: MongoDB (via Mongoose schemas) storing securely hashed biometric records, active transaction ledgers, and career milestones.

## 💎 Core Modules & User Flows (How They Work Under the Hood)

### 📈 1. Main Command Center (Dashboard)
* **What it is**: The user's welcome launchpad.
* **Key Features**: Features interactive **Spline 3D Scenes** and **Parallax Scroll Sections** that dynamically walk the user through the system pillars.
* **Digital Twin Today Portal**: At the bottom of the dashboard, a sleek call-to-action directs users to the unified Digital Twin Today page.

### 👥 2. Your Digital Twin Today Page (`/goals`)
* **What it is**: The central hub that gives a comprehensive overview of the virtual clone's state.
* **Key Features**:
  1. **Scores Overview**: Live telemetry scoring showing Health, Financial, and Career Progress indices.
  2. **Active Goal Trajectories**: Responsive progress meters demonstrating objectives and completion deadliness.
  3. **AI Recommendations**: Actionable alerts generated by the twin's predictive model with quick action buttons (e.g., "Log Hydration", "Run Simulator").
  4. **Activity Ledger Timeline**: A chronological ledger auditing all local synchronization event cards logged today.

### 🧬 3. Health Console (`/health`)
* **What it is**: Biometric ledger tracking caloric load, sleep cycles, and hydration.
* **Key Features**:
  * **Frictionless Numeric Inputs**: Direct numerical forms carefully bound to React state so that users can type and backspace without input lockups or lag.
  * **Routine Presets**: Quick choice chips (e.g., *Early Morning Run*, *Cheat Breakfast*, *Desk Focus*, *Deep Sleep*) that instantly auto-populate typical metric combinations.
  * **AI Auto-Gen Routine**: A randomized biometric compiler that constructs natural-looking mock daily logs with realistic variations.
  * **Sleep Architecture History**: An area trend chart showing total sleep duration across the past seven logged database entries.

### 🧠 4. AI Simulation Engine (`/simulator`)
* **What it is**: The core scenario planning panel.
* **Key Features**:
  * **Parameter Tuning**: Sliders to adjust daily parameters like Exercise Frequency, Study Hours, Savings Rates, and Sleep Allocations.
  * **Robust Multi-Model Fallback AI Loop**: 
    * To ensure the simulation always runs—even during API rate limits or service outages—the backend server implements a fallback loop.
    * It sequentially requests predictions from **Google Gemini 2.5 Flash**, **Llama 3 8B**, and **GPT-3.5**. If a model fails or encounters an error, it silently fails over to the next one within milliseconds, guaranteeing unbroken simulation uptime.

### 🏆 5. Gamification & Achievements (`/achievements`)
* **What it is**: System for keeping users engaged and consistent.
* **Key Features**:
  * **XP & Level Progress**: Dynamic calculation of levels and progress meters based on database consistency.
  * **Grayscale-to-Glow Badging**: Locked badges are rendered in grayscale glass containers which transition into glowing violet-bordered badges once achieved.

## 🏃 Setup & Launch Instructions

To demonstrate or run the project locally, launch both the backend and frontend:

### 1. Spin Up the Backend Server
```bash
cd backend
npm install
npm run dev
```
*Runs on `http://localhost:5000` with active MongoDB connections.*

### 2. Launch the Vite Frontend
```bash
cd frontend
npm install
npm run dev
```
*Vite compiles and serves the application on `http://localhost:5173`.*
